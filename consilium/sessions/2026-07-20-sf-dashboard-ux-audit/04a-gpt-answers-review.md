# GPT — ревью Answers-first прототипа

**Оценка:** концепция 8/10, текущая UX/UI-реализация ~7/10. «Концептуально это уже
правильный продукт, а не косметический редизайн старого дашборда… но пока остаётся
слишком текстовым и вертикальным».

## Что работает

1. Answers-first считывается — первый экран сразу отвечает.
2. Структура «вывод → доказательство → действие», язык клиента.
3. Честность данных как часть интерфейса (confidence, база, Insufficient data).
4. Доказательства встроены в сценарий.
5. Навигация стала клиентской; «Справочник» правильно отделяет reference.

## P0

1. **Executive summary сверху:** четыре компактные карточки (вопрос · короткий
   ответ · надёжность), клик скроллит к подробному ответу. Страница = executive
   summary + подробный отчёт одновременно.
2. **Вопрос 1 переклеймлен:** «тема живая и устойчиво активная» — вывод об
   информационной активности, не о спросе (клиники не отделены). Разделить:
   «Активность темы — Medium», «Рыночный спрос — Insufficient data»; вердикт —
   «активность стабильная… реальный пациентский спрос пока нельзя надёжно оценить».
3. **17,1 млн — не «просмотры»:** природа Views неизвестна → «оценочный
   потенциальный охват» + tooltip про методику.

## P1

- Карточки перегружены (9 уровней) → структура: вопрос · ответ одним предложением ·
  2–3 цифры · одна визуализация · рекомендация · evidence+methodology. Формулировку
  из брифа — под tooltip.
- График динамики декоративный → значения в tooltip, подпись пика, маркировка
  неполного месяца, средняя линия, нейтральный заголовок «Объём сообщений по
  месяцам»; без намёка на рост при отсутствии прошлого периода.
- Media Types — доли, не только абсолюты (846 · 46 %).
- Sentiment: не смешивать нейтральные и неразмеченные; если данные не позволяют —
  «Отношение не определено — 1 045», а не «нейтральные».
- «6:1» сильнее базы → «Среди сообщений с определённым отношением позитивные в
  6 раз чаще негативных» + «База: 794 из 1 839 · 43 %» рядом.
- Демография при 7,7 % — не визуализировать барами: спец-состояние «Недостаточно
  данных», цифры 122/19 — в раскрываемую методологию.
- Confidence локализовать: «Надёжность вывода: средняя/низкая», «Недостаточно
  данных» + tooltip про факторы оценки.
- «Что это значит» → прикладные «Рекомендации» (канал — видео и соцсети; работать
  со страхами, а не с «негативной репутацией»).

## P2

- Рабочая область до 960–1080 px (двухколоночная компоновка вывод/график + supporting).
- Высоту карточек −15–20 %.
- CTA: «5 сообщений, подтверждающих вывод →», «Посмотреть 114 негативных →» +
  вторая ссылка «Методика и ограничения».
- Cyan слишком часто: cyan — ключевой ответ и навигация; нейтральный синий —
  графики; зелёный — подтверждённый позитив; жёлтый — ограничения; красный —
  негатив/критично низкая надёжность.

## Финальная структура страницы

1. Executive summary (4 карточки) → 2. Подробные ответы → 3. Общий methodology
footer (период · источники · что считается сообщением · как рассчитан охват ·
классификации · ограничения).

**Главный вывод ревью:** следующая итерация — не полировка, а разделение уровней
(сверхкороткие ответы для руководителя ↑, доказательства ↓). Критично не превращать
честную аналитику обратно в маркетинговые обещания: активность ≠ спрос, оценочный
охват ≠ просмотры, 141 сообщение ≠ демография рынка.
